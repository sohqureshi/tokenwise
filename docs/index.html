⚠️ The Problem

❌ Raw JSON increases token cost
❌ Extra structure wastes LLM budget
❌ Slower responses from large payloads

🚀 The Solution

TokenWise intelligently reduces LLM payload size by pruning, compressing, and restructuring JSON before it reaches the model.

Less noise → fewer tokens → lower API cost → faster responses.

💻 CLI Usage

Optimize any JSON file directly from your terminal: 

node cli/index.js optimize input.json

Tip: Works best with nested API payloads or LLM prompts.

📉 Transform Examples

prune()

Removes irrelevant or noisy fields

{
  "user": {
    "id": 1,
    "name": "John",
    "debug": true,
    "internal": "cache-data"
  }
}
{
  "user": {
    "id": 1,
    "name": "John"
  }
}
compact()

Minimizes structure without changing meaning

{
  "user": {
    "profile": {
      "name": "John"
    }
  }
}
{"user":{"profile":{"name":"John"}}}
flatten()

Converts nested objects into key paths

{
  "user": {
    "profile": {
      "name": "John"
    }
  }
}
{
  "user.profile.name": "John"
}
toNatural()

Converts JSON into LLM-friendly text

{
  "user": {
    "name": "John",
    "theme": "dark"
  }
}
User John prefers dark theme.
tokenEstimate()

Estimates token cost before sending to LLM

{"text": "Hello world"}
~3–5 tokens
